# Архитектура системы

## Стек

| Слой | Технология |
|---|---|
| Backend | Python 3.11 + FastAPI |
| AI-агенты | Claude API (claude-sonnet-4-6) |
| Парсинг МК | pdfplumber (без AI) |
| Парсинг чертежей | Claude Vision API |
| Поиск поставщиков | Tavily API + собственная БД |
| БД | PostgreSQL (Supabase) |
| Frontend | React + Vite + TailwindCSS |
| Desktop | Electron (обёртка frontend) |
| Экспорт | openpyxl (Excel) |

## Агенты

| Агент | Вход | Выход |
|---|---|---|
| Проверяющий | МК / чертёж | Статусы полей (missing / extracted / confirmed) |
| Расчётчик | Подтверждённый BOM | Snapshot себестоимости |
| Закупщик | BOM + склад + регион | purchase_request + поставщики + RFQ |
| Сравнитель | КП от поставщиков | Взвешенный скоринг + рекомендация |

## Поток данных

```
Заказ → проверка изделия в справочнике
  → [новое изделие] загрузка чертежа → AI-парсинг → подтверждение → нормы
  → [известное] берём нормы из product_materials
        ↓
МК → pdfplumber парсинг → field_status на каждое поле
        ↓
Агент-Проверяющий → missing блокирует → пользователь подтверждает
        ↓
Агент-Расчётчик → BOM + snapshot цен + себестоимость
        ↓
Проверка stock_balances → purchase_request
  → qty_in_stock покрыто складом
  → qty_to_purchase → rfq_items
        ↓
Агент-Закупщик → своя БД поставщиков → Tavily если мало → RFQ письма
        ↓
Агент-Сравнитель → score_total (цена 40% + срок 25% + верификация 15% + НДС 10% + тип 10%)
        ↓
Менеджер выбирает → Excel / письмо поставщику
```
